What are the 4 Report Formats in Salesforce — when do you use each?REPORTS
  • Tabular: Simple list of records. No groupings. Like an Excel sheet. (e.g., "Mailing list of all borrowers").
  • Summary: Groupings by rows. Includes subtotals. (e.g., "Loans grouped by Status").
  • Matrix: Groupings by both rows AND columns. (e.g., "Loans grouped by Status (rows) and Month (columns)").
  • Joined: Combine multiple report blocks (different report types) in one view. (e.g., "Show Account details in Block A and related Opportunities in Block B").
Dashboard Tip

Only Summary and Matrix reports can be used as source data for dashboard components (charts, gauges).

Report Types — standard vs custom, joined reports, cross-filters?REPORTS MASTER

Custom Report Types (CRT) — define which objects and fields are available in a report. Necessary when standard report types don't include the fields or relationships you need.

  • Create CRT — Setup → Report Types → New. Define primary object, related objects, and which fields are available
  • One-to-many relationship — CRT can include parent + child in same report (Account with Loans)
  • Field availability — CRT controls which fields users see in the report builder — useful for hiding sensitive fields from non-admin report builders
  • Joined Reports — combine up to 5 report blocks with different report types. Each block is a separate dataset. Can show side-by-side comparisons (Loans by Status this quarter vs same quarter last year)
  • Cross-filters — filter a report based on related object criteria without showing the related object's fields. Example: Accounts WITH active Loans (without adding loan fields to the report)
// Access Reports via Apex Analytics API
Reports.ReportResults results = Reports.ReportManager.runReport('00O000000000001AAA', true); // true = detail rows
Reports.ReportMetadata meta = results.getReportMetadata();
Reports.ReportFactWithDetails facts = (Reports.ReportFactWithDetails) results.getFactMap().get('T!T');

for (Reports.ReportDetailRow row : facts.getRows()) {
  List<Reports.ReportDataCell> cells = row.getDataCells();
  String loanName   = (String) cells[0].getValue();
  Decimal amount    = (Decimal) cells[1].getValue();
}
Dynamic Dashboards and Dashboard Filters — how do they work?REPORTS MASTER
  • Dynamic Dashboards — show data as the viewing user (respects their sharing and visibility). Standard dashboards show data as the dashboard's running user (one fixed view for all).
  • When to use Dynamic — each user should see only their data (loan officers see their own loans). When to use Standard — executive dashboards where everyone needs the same org-wide view.
  • Dashboard Filters — add up to 3 filter options users can toggle without editing the dashboard. Example: filter by Branch, by Loan Type, by Date Range
  • Scheduled Reports — Setup → Report → Schedule. Sends report to email recipients on a schedule. Maximum 200 scheduled reports per org.
  • Analytics API — Reports.ReportManager.runReport() runs synchronously for <2,000 rows. Use Reports.ReportManager.runAsyncReport() for larger datasets.
Dashboard component limit

Each dashboard supports up to 20 components. For complex executive dashboards, use Analytics Studio (Einstein Analytics/CRM Analytics) which has no such limit and supports more chart types.
