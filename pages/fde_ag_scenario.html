Practical challenges in designing, building, and debugging Salesforce Agents
Scenario: Agent is hallucinating or giving wrong answers. How do you debug?DEBUGΓû╝
  1. Review Conversation Logs: Look at the exact prompt sent to the LLM.
  2. Check Grounding: Was the correct record data retrieved? If not, check Action inputs.
  3. Refine Instructions: Is the Topic description too vague? Atlas might be routing to the wrong topic.
  4. PII Masking check: Did the Trust Layer mask a critical piece of info needed for the answer?
Scenario: Design an Agent for a Mortgage Borrower checking loan status.DESIGNΓû╝
  • Topic: "Loan Status Inquiries"
  • Action: Apex @InvocableMethod `getLoanDetails(loanNumber)`
  • Grounding: Ground the prompt with Loan__c fields (Status, CloseDate).
  • Guardrails: "If loan is 'Declined', do not give reasons. Escalate to Loan Officer."
