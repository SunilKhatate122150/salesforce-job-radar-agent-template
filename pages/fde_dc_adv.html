Mortgage company has data in CRM, legacy LOS, and payments system. Build a unified profile.SCENARIO▼
  • Step 1: CRM data via Salesforce CRM Connector — Contact, Account, Opportunity → Individual DMO + Loan_Application DMO
  • Step 2: LOS data via S3 batch connector — nightly export; DLO → Loan_Account DMO + Individual DMO
  • Step 3: Payments via MuleSoft connector or Streaming API → Payment_Engagement DMO
  • Step 4: Identity Resolution — primary: exact email; secondary: SSN last-4 + DOB. Reconciliation: most recent wins for address, CRM wins for name
  • Step 5: Calculated Insights — total balance, credit utilization, days since last payment, LTV ratio
  • Step 6: Sync enriched profile back to CRM Contact for Sales/Service and Agentforce grounding

How would you use Data Cloud to trigger a real-time journey when a borrower's ARM rate is about to adjust?SCENARIO▼
  • Rate adjustment date stored in Loan_Account DMO. Create Calculated Insight: days_until_rate_adjustment
  • Build Streaming Insight rule: when days_until_rate_adjustment = 90, fire trigger event
  • Activation → Marketing Cloud Journey Builder: "ARM Refinance Outreach" journey entry
  • Journey sends personalized email → if no response in 7 days → escalates to Sales Cloud task for loan officer
  • Agentforce agent also primed: unified profile enriched with "rate adjusts in 90 days" → agent proactively mentions it
End-to-end

This shows Data Cloud → Marketing Cloud → Sales Cloud → Agentforce working together. Exactly what FDE interviews reward.

Duplicate Unified Individuals in Data Cloud. How do you investigate and fix?ARCHITECTURE▼
  • Step 1: Check Identity Resolution job logs — did the job run? Are match rules processing all DMOs?
  • Step 2: Match rules too narrow — only exact email misses records where email is blank
  • Step 3: Data quality — "John@bank.com" vs "john@bank.com" breaks exact matches without normalization
  • Step 4: Add secondary match rule — phone, SSN hash, or customer ID
  • Step 5: Review reconciliation rules — ensure merge logic is intentional
  • Step 6: Re-run Identity Resolution — it rebuilds all Unified Individuals from scratch
How does Data Cloud help Agentforce give personalized responses to a returning mortgage customer?SCENARIO▼
  • Without Data Cloud: agent only has CRM data — limited to one system's view
  • With Data Cloud: agent retrieves Unified Individual — includes CRM, LOS, payments, web behavior across ALL touchpoints
  • Grounding prompt becomes richer: "This customer has 2 active loans, logged in 3 days ago, ARM adjusts in 87 days, LTV is 72%"
  • Instead of generic "How can I help?", agent says: "Welcome back. Your adjustable rate is coming up for review — would you like to explore refinancing?"

This is the Agentforce + Data Cloud core value — grounding enables personalization at scale without human involvement.

What are Data Graphs in Data Cloud? Why critical for Agentforce?CONCEPT▼
  • Data Graphs = pre-built, optimized views of related Data Cloud objects in a single retrievable entity
  • Think of them as materialized views — pre-joined data for fast retrieval
  • Without: Agent runs 5 separate queries → slow, multiple round-trips
  • With: One query returns complete customer profile → sub-second retrieval
  • Created in: Data Cloud Setup → Data Graphs → define root entity + related entities
Performance insight

"Data Graphs make Agentforce grounding fast at scale — pre-join data so the agent doesn't pay query cost at conversation time."

What is Data Cloud's Vector Database / Embeddings? How does RAG actually work?ARCHITECTURE▼

Data Cloud has a native vector database for storing embeddings (numerical representations of text).

RAG Flow: 1. Upload knowledge docs (policy PDFs, FAQ) to Data Cloud 2. Einstein generates embeddings for each document chunk 3. User asks: "What are jumbo loan income requirements?" 4. Query is EMBEDDED → numerical vector 5. Vector search finds top matching doc chunks by similarity 6. Matching chunks injected into agent prompt as context 7. LLM responds from YOUR docs — not training data Result: Accurate, sourced answer from authoritative content

Mortgage use: Upload compliance handbook → agent retrieves relevant section dynamically instead of hardcoding every answer.

Key definition

"RAG is an architecture pattern connecting generative AI to authoritative knowledge. Vector search is the retrieval mechanism."

What is the Zero Copy Partner Network? Why is it significant?CONCEPT▼
  • Zero Copy = data accessed in-place without physically copying to Data Cloud
  • Partners: Snowflake, Google BigQuery, Databricks, Amazon Redshift
  • Bidirectional live query federation — Data Cloud queries Snowflake in-place AND vice versa
  • Benefits: No duplication (cost), no ETL (speed), data stays governed at source (compliance)
  • Mortgage: Core banking data warehouse in Snowflake queried without copying billions of records
Architecture position

"Zero Copy is Salesforce's answer to the lakehouse architecture — federate queries instead of moving data."
