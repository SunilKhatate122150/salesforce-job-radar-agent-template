Relationship Name: Standard objects often use the plural name (Contacts, Opportunities). Custom objects use the relationship name defined on the Lookup/Master-Detail field + __r. If you are unsure, find the field in Setup → Field Name → Child Relationship Name.

Query parent data from a child (Child-to-Parent) — get Loan Officer name from Loan record?SOQL

Use dot-notation in the SELECT clause. You can navigate up to 5 levels up. For custom objects, use __r instead of __c for the parent reference.

Why use these over regular subqueries?
Regular subqueries (SELECT ... (SELECT...)) include the child data in the result set, consuming more memory. Semi-joins and Anti-joins filter the parent list WITHOUT returning child data, which is faster and cleaner when you don't actually need to process the child records.

What is SOSL (Salesforce Object Search Language) — when to use it over SOQL?SOSL


SOSL is used for keyword-based search across multiple objects at once. It is faster than SOQL when searching across text fields and when the field you are searching is not indexed.


// SOSL search for "Smith" across Leads, Contacts, and Loans

How do you implement Dynamic SOQL — security best practices (SQL Injection)?SOQL

Dynamic SOQL is built as a string at runtime. You MUST use bind variables (preferred) or String.escapeSingleQuotes() to prevent SQL Injection.

// DYNAMIC SOQL with Bind Variables (Safe & Clean)
String status = 'Approved';
String query = 'SELECT Id, Name FROM Loan__c WHERE Status__c = :status';
List<Loan__c> loans = Database.query(query);

// DYNAMIC SOQL with string building (Danger — needs escaping!)
String userInput = 'FHA';
String safeInput = String.escapeSingleQuotes(userInput);
String query2 = 'SELECT Id FROM Loan__c WHERE LoanType__c = \'' + safeInput + '\'';
List<Loan__c> results = Database.query(query2);
