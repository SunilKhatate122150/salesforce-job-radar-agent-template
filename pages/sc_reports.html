How do you read a Salesforce report from Apex using the Reports API?REPORTS▼
// Synchronous report run — returns up to 2000 rows, immediate response
Reports.ReportResults results = Reports.ReportManager.runReport('00O5g...', true);
// true = include detail rows; false = summary only (faster)

// Access the fact map
// Key format: 'rowGroupIndex!colGroupIndex' or 'T!T' for grand total
Reports.ReportFactWithDetails grandTotal =
    (Reports.ReportFactWithDetails) results.getFactMap().get('T!T');

Integer rowCount = grandTotal.getRows().size();
for(Reports.ReportDetailRow row : grandTotal.getRows()) {
    List<Reports.ReportDataCell> cells = row.getDataCells();
    String loanName = cells[0].getLabel();   // formatted display value
    Object amount = cells[1].getValue();     // raw value (Decimal, Id, String)
    System.debug(loanName + ': ' + amount);
}

// Get column names from metadata
List<String> columns = results.getReportMetadata().getDetailColumns();

// Override filters at runtime (no change to saved report)
Reports.ReportMetadata meta = Reports.ReportManager.describeReport('00O...').getReportMetadata();
Reports.ReportFilter filter = new Reports.ReportFilter();
filter.setColumn('STATUS');
filter.setOperator('equals');
filter.setValue('Active');
meta.setReportFilters(new List<Reports.ReportFilter>{ filter });
Reports.ReportResults filtered = Reports.ReportManager.runReport(meta, true);

// Asynchronous run (for reports over 2000 rows)
Reports.ReportInstance instance = Reports.ReportManager.runAsyncReport('00O...', true);
String instanceId = instance.getId();
// Poll: Reports.ReportManager.getReportInstance('00O...', instanceId).getStatus()
What is a Dynamic Dashboard — how does it work and what are the limitations?REPORTS▼

A Dynamic Dashboard uses "Run As: Logged-in user" instead of a fixed user. Each person who opens the dashboard sees data filtered by their own permissions and record ownership — everyone sees their own data from the same dashboard.

Mortgage use case: One "Loan Officer Performance Dashboard" showing My Active Loans, My Pipeline Value, My Closing This Week. Loan Officer A sees their 15 loans. Officer B sees their 8 loans. Admin sees all.

  • Cannot be scheduled for email refresh (only static dashboards can be scheduled)
  • Cannot be combined with Dashboard Filters
  • Maximum 10 dynamic dashboards per org (varies by edition)
  • Underlying reports must use "with sharing" data — no system-mode access
  • Each viewer must have access to the underlying report
What is a Cross Filter in a report — show a mortgage example?REPORTS▼

Cross Filters let you filter report results based on whether related records EXIST or DON'T EXIST. They answer questions that regular filters cannot — "Show me Loans that have NO uploaded appraisal document."

In Report Builder: Add Filter → Cross Filter → "[Object] with/without [Related Object]"

  • "Loans WITHOUT ContentDocumentLink where Title contains 'Appraisal'" — loans missing appraisal
  • "Applications WITHOUT Loan__c where Status = Active" — applications with no active loans
  • "Contacts who HAVE Loan_Borrower__c where Loan Status = Funded" — all funded borrowers
  • "Accounts WITHOUT Opportunities WHERE CloseDate is this year" — no closed business this year
Compliance use case

For TRID compliance: "Show all Loans where ClosingDate is in next 3 days WITHOUT a ContentDocumentLink where Title = 'Closing Disclosure'" — loans approaching closing without the required CD document uploaded. This becomes a daily compliance report.

How do you run a Salesforce report from Apex and process the results?REPORTS▼
// Run report synchronously (max 2,000 rows)
Reports.ReportResults results = Reports.ReportManager.runReport('00O5g000004xyzA', true); // true = detail rows

Reports.ReportFactWithDetails facts = (Reports.ReportFactWithDetails) results.getFactMap().get('T!T');
List<Reports.ReportDetailRow> rows = facts.getRows();

for (Reports.ReportDetailRow row : rows) {
  List<Reports.ReportDataCell> cells = row.getDataCells();
  String loanName   = (String)  cells[0].getValue();
  Decimal amount    = (Decimal) cells[1].getValue();
  String status     = (String)  cells[2].getValue();
  System.debug(loanName + ': $' + amount + ' (' + status + ')');
}

// Run asynchronously (for reports >2,000 rows)
String jobId = Reports.ReportManager.runAsyncReport('00O5g000004xyzA', true);
// Poll for completion:
Reports.ReportInstance instance = Reports.ReportManager.getReportInstance(jobId);
while (instance.getStatus() != 'Success') {
  // wait and poll (in practice, use a scheduled job to check)
  instance = Reports.ReportManager.getReportInstance(jobId);
}
Reports.ReportResults asyncResults = instance.getReportResults();
How do you build a report that shows data from custom objects with complex relationships?REPORTS▼

When standard report types don't include the object or relationship you need, create a Custom Report Type (CRT).

  • Step 1 — Setup → Report Types → New Custom Report Type
  • Step 2 — Select primary object: Loan__c
  • Step 3 — Add related objects: Account (borrower), Loan_Document__c (documents), Loan_Officer__r (owner's User record)
  • Step 4 — Choose relationship type: 'A may or may not have B' (outer join — shows loans even without documents)
  • Step 5 — Add fields to layout: drag all fields you want available in reports to the field layout
  • Step 6 — Publish and assign to appropriate profiles
Outer vs Inner join

'A must have B' = inner join — only records with a related record appear. 'A may or may not have B' = outer join — all primary records appear. For reports showing loans regardless of whether they have documents, use 'may or may not have'.

What are Bucket Columns in reports — when are they more useful than formula fields?REPORTS▼

Bucket Columns let you categorise field values into groups directly in the report — without creating a formula field on the object. Available for number, text, and picklist fields.

  • Use Bucket for — ad-hoc grouping that only applies to this report. No object schema change needed.
  • Example: Loan Amount Buckets — Small (<100K), Medium (100K–500K), Large (500K+)
  • Example: Credit Score Tiers — Excellent (750+), Good (700-749), Fair (620-699), Poor (<620)
  • Use Formula Field instead when — grouping is used in many reports, needs to appear on record page, or the calculation is complex
// Bucket Column config in Report Builder (UI only — not in Apex):
// Field: Amount__c
// Bucket name: Loan Size
// Rule 1: Amount__c < 100000 → 'Small'
// Rule 2: Amount__c >= 100000 AND <= 500000 → 'Medium'
// Rule 3: Amount__c > 500000 → 'Large'
// After bucketing: can group by Loan Size and show count/sum per bucket
How do you create a dashboard that refreshes on a schedule and emails to stakeholders?REPORTS▼
  • Dashboard refresh — Dashboards refresh automatically when viewed if last refresh >24h old. For scheduled refresh: Setup → Dashboard → Subscribe → set schedule
  • Subscribe to dashboard — users can subscribe to receive dashboard snapshots via email. Set frequency: daily/weekly. Sends a PDF snapshot of the dashboard.
  • Report subscriptions — From Report → Subscribe → set schedule and email format. Can send CSV, Excel, or formatted email. Filter conditions: only send if results meet criteria (e.g., only alert if loans past due > 0)
  • Conditional alerts — Reports → Subscribe → Add Filter: 'Run report only if row count > 0'. Useful for exception reporting — only email when there is something to act on.
  • Salesforce Flow scheduled reports — create a Scheduled Flow that calls a Report in Apex and emails results in a custom format with additional business logic
Dashboard data is cached

Dashboards show data from when they were last refreshed — not real-time. If a stakeholder needs live data, point them to the underlying report (always shows current data) or use a custom LWC dashboard connected to @wire.

Real-timeYes — queries liveNear-real-time — data synced via dataflow
External dataNoYes — connect to external databases, CSV, Salesforce
AI/MLNoBuilt-in predictions, Einstein Discovery
ComponentsCharts, tables, filtersLenses, dashboards, stories, Einstein Predictions
CostIncluded in SalesforceRequires separate license

When reporting on tens of millions of records, when you need cross-cloud analytics (Salesforce + external data warehouse), when stakeholders want interactive 'slice and dice' exploration rather than static reports.
