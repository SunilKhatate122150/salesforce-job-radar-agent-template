How do you trigger an Agentforce agent from LWC — can you embed it?AGENTFORCE

You don't "call" an agent via API normally; you use the standard Lightning Messaging for Web/In-App component. The agent is the backend engine. However, you can use the lightning-conversational-agent-service to interact with an active session.

// Communicate with an active agent session in LWC
import { LightningElement, wire } from 'lwc';
import { getAgentSession } from 'lightning/conversationalAgentService';

export default class AgentIntegrator extends LightningElement {
    @wire(getAgentSession)
    session;

    handleSendMessage() {
        if(this.session.data) {
            // Programmatically send a message as if the user typed it
            this.session.data.sendMessage('Check my loan status for application 78542');
        }
    }
}
What is a Prompt Template and how does it work with Agentforce?AGENTFORCE

Prompt Templates are where you define the "personality" and "logic" of an AI response. They are stored as metadata. You can include record data directly in the prompt using {!$Input:Loan__c.Name}.

  • Salesforce Prompt Builder: Declarative tool to write prompts.
  • Grounding: Drag and drop record fields or related lists into the prompt.
  • Apex Grounding: Use {!$Apex:MyGroundingClass.getMethod()} to inject complex logic into the prompt at runtime.
  • Agent Integration: The agent calls a "Prompt Action" which executes the template and returns the LLM response to the user.
Developer Task

Your job isn't just to write the Apex; it's to write the "System Prompt" — the instructions that tell the AI how to behave. "You are a senior mortgage underwriter. Summarize this loan application focusing on credit risk. Use a professional but empathetic tone."

How do you handle multi-step actions in Agentforce (e.g., lookup → calculate → update)?AGENTFORCE

The "Atlas" reasoning engine handles the sequencing. You don't write one massive Apex class; you write small, atomic actions.

  1. Action 1 (Apex): getLoanDetails(loanId) — returns the record.
  2. Action 2 (Apex): calculatePayment(amount, rate) — returns a number.
  3. Action 3 (Flow/Apex): updateLoanEstimate(loanId, payment) — performs the update.

The Agent reads the user's request: "Update my loan estimate based on a 7% rate." Atlas decides: "I need to get details first, then calculate, then update." It calls your actions in that order automatically.

What are the security implications of Agentforce — does it respect sharing rules?AGENTFORCE
  • User Context: Agentforce actions run in the context of the Running User. If the user doesn't have access to a record via sharing rules, the agent's Apex action (if written correctly with WITH USER_MODE) won't see it either.
  • Prompt Data: Only data the user is allowed to see is injected into prompt templates.
  • Trust Layer: As discussed, PII is masked before being sent to the LLM.
  • Permission Sets: You must assign the "Einstein Agent" or "Einstein Copilot" permission set to users for them to interact with the agent.
Security Warning

If you write an Apex action with without sharing, the agent could potentially reveal information to a user they aren't supposed to see. ALWAYS use with sharing or inherited sharing for Agentforce actions.

How do you debug an agent that is "going off track" or hallucinating?AGENTFORCE
  • Conversation Logs: Check the exact transcript. See what the "System Prompt" was and what data was grounded.
  • Agent Builder Simulator: Use the 'Show Runtime Details' toggle. It shows the agent's "Thought Process" — why it chose a specific action.
  • Topic Guardrails: Refine the instructions. Add "Do NOT" statements. "Do not speculate on interest rates. Only provide the rates returned by the GetRates action."
  • Negative Testing: Try to "jailbreak" your agent by asking out-of-scope questions. Ensure it gracefully declines or escalates.
What is the "Atlas" engine — explain it to an architect?AGENTFORCE

Atlas is the orchestration layer between the user's natural language and your Salesforce logic. It replaces the old "if/then" branching of traditional bots with an autonomous reasoning loop.

  1. Understand: Uses LLM to parse intent and extract entities (e.g., Loan ID).
  2. Plan: Identifies which "Topics" and "Actions" are relevant.
  3. Act: Executes the chosen Apex or Flow action.
  4. Observe: Reviews the data returned by the action.
  5. Reason: Decides if the goal is met or if another step is needed.
  6. Respond: Generates the final natural language answer.
How do you deploy Agentforce configurations between environments?AGENTFORCE
  • Metadata Types: Bot (the agent definition), GenAiPromptTemplate, GenAiPlanner, InvocableAction.
  • Change Sets / DevOps Center: You can include these metadata types in your deployment.
  • Post-Deployment: Some manual activation is usually required in the target org (Setup → Agent Builder → Activate).
  • API Name Sync: Ensure your Apex classes and Flows have identical API names across environments so the agent's action mappings don't break.
