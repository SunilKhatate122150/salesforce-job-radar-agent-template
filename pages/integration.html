How do you make an authenticated REST API callout in Apex? Show complete code with Named Credentials. INTEGRATION 
// Named Credential: Created in Setup ΓåÆ Security ΓåÆ Named Credentials
// Name: ExperianCreditAPI, URL: https://api.experian.com
// Auth: OAuth 2.0 configured here (no credentials in code)

public class ExperianCreditService {
    
    public static CreditResponse pullCreditReport(String ssn, String lastName) {
        
        // Build request using Named Credential (callout:ExperianCreditAPI)
        HttpRequest req = new HttpRequest();
        req.setEndpoint('callout:ExperianCreditAPI/v1/credit-report'); // Named Cred prefix
        req.setMethod('POST');
        req.setHeader('Content-Type', 'application/json');
        req.setHeader('Accept', 'application/json');
        req.setTimeout(30000); // 30 seconds max
        
        // Build request body
        Map<String, Object> body = new Map<String, Object>{
            'ssn' => ssn,
            'lastName' => lastName,
            'requestType' => 'FULL'
        };
        req.setBody(JSON.serialize(body));
        
        // Execute callout
        Http http = new Http();
        HttpResponse res;
        
        try {
            res = http.send(req);
        } catch(CalloutException e) {
            throw new CreditServiceException('Callout failed: ' + e.getMessage());
        }
        
        // Handle response by status code
        if(res.getStatusCode() == 200) {
            return (CreditResponse) JSON.deserialize(res.getBody(), CreditResponse.class);
        } else if(res.getStatusCode() == 401) {
            throw new CreditServiceException('Authentication failed - check Named Credential');
        } else if(res.getStatusCode() == 429) {
            throw new CreditServiceException('Rate limit exceeded - retry after delay');
        } else {
            throw new CreditServiceException('API Error ' + res.getStatusCode() + ': ' + res.getBody());
        }
    }
    
    // Response wrapper class
    public class CreditResponse {
        public Integer ficoScore;
        public String riskCategory;
        public List<TradelineItem> tradelines;
    }
}

Why can't you make callouts from triggers? How do you work around it? INTEGRATION

Triggers run synchronously within a database transaction. If a callout is made during a transaction and the external system takes 30 seconds, the database transaction locks those records for 30 seconds — this is a major problem in a multitenant environment.

Salesforce enforces: You cannot make an HTTP callout from within a trigger (or any code that runs in a transaction that has already performed DML).

// This throws: System.CalloutException: You have uncommitted work pending
trigger LoanTrigger on Loan__c (after insert) {
    for(Loan__c loan : Trigger.new) {
        ExperianService.pullCredit(loan.Id); // ERROR - DML already happened
    }
}

// SOLUTION 1: @future with callout=true
trigger LoanTrigger on Loan__c (after insert) {
    Set<Id> loanIds = Trigger.newMap.keySet();
    ExperianService.pullCreditAsync(loanIds); // fires after transaction commits
}

public class ExperianService {
    @future(callout=true)
    public static void pullCreditAsync(Set<Id> loanIds) {
        // This runs AFTER the trigger transaction completes
        // Safe to make callouts here
        for(Id loanId : loanIds) {
            pullCreditReport(loanId); // callout now safe
        }
    }
}

// SOLUTION 2: Queueable (better - sObject support, chainable)
trigger LoanTrigger on Loan__c (after insert) {
    System.enqueueJob(new CreditCheckQueueable(Trigger.new));
}
