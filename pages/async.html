// @future - simple, no sObject params, not chainable
@future(callout=true)
public static void sendToExternalSystem(List<Id> recordIds) {
    // Only Id (primitive) - not full sObject
    List<Loan__c> loans = [SELECT Id, LoanAmount__c FROM Loan__c WHERE Id IN :recordIds];
    // Make callout here
}

// Queueable - complex async, chainable, sObject allowed
public class CreditCheckQueueable implements Queueable, Database.AllowsCallouts {
    private List<Loan__c> loans;
    public CreditCheckQueueable(List<Loan__c> loans) { this.loans = loans; }
    
    public void execute(QueueableContext ctx) {
        // Process loans, make callout to Experian
        // Chain another job if needed:
        System.enqueueJob(new OFACCheckQueueable(loans));  // chaining!
    }
}
// To start: System.enqueueJob(new CreditCheckQueueable(loans));

// Batch - large data volumes
public class LoanStatusBatch implements Database.Batchable<SObject>, Database.Stateful {
    public Integer successCount = 0; // Stateful - preserved between chunks
    public Database.QueryLocator start(Database.BatchableContext bc) {
        return Database.getQueryLocator('SELECT Id, Status__c FROM Loan__c WHERE Status__c = \'Active\'');
    }
    public void execute(Database.BatchableContext bc, List<SObject> scope) {
        // Process up to 200 records per chunk
        successCount += scope.size();
    }
    public void finish(Database.BatchableContext bc) {
        System.debug('Total processed: ' + successCount);
    }
}
// To run: Database.executeBatch(new LoanStatusBatch(), 200);
What is Mixed DML error and how do you fix it? ASYNC

Mixed DML occurs when you try to do DML on a setup object (like User, Profile, PermissionSet) AND a non-setup object (like Account, Contact) in the same transaction. Salesforce does not allow this.

Setup objects: User, UserRole, Profile, PermissionSet, PermissionSetAssignment, GroupMember, etc.

// This throws: MIXED_DML_OPERATION error
User u = new User(ProfileId = adminProfileId, Username = 'email-protected'...);
insert u;  // Setup object DML

Account a = new Account(Name = 'Test');
insert a;  // Non-setup DML in same transaction = ERROR

// FIX 1: Use @future for the non-setup DML
@future
public static void createAccountAsync(String name) {
    insert new Account(Name = name);
}
// @future runs in a NEW transaction - no Mixed DML

// FIX 2: In tests, use System.runAs() to separate transactions
@isTest
static void testUserCreation() {
    User newUser; 
    System.runAs(new User(Id = UserInfo.getUserId())) {
        newUser = createTestUser(); // Setup DML here
    }
    System.runAs(newUser) {
        insert new Account(Name = 'Test'); // Non-setup DML here
    }
}
Real scenario from your projects

In the Mortgage platform, when creating a new loan officer (User) and their first Loan record, you'd hit Mixed DML. Solution: insert User first, then use @future to create the Loan record in a separate transaction.
